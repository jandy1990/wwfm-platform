1. Overview & Core Concepts

Page Purpose

The goal page is the heart of WWFM - where users discover solutions that actually worked for specific life challenges. It must balance information density with visual clarity, making it easy to scan solutions while providing transparency about user-generated data.

Key Design Principles

🎯 Solutions First

Solutions are visible within 2 seconds of page load. No hero images, no fluff - straight to what worked.

📊 Transparent UGC

Users can see that data comes from real people through report counts and distribution breakdowns.

🔄 Progressive Disclosure

Simple view for scanning, detailed view for research. User controls the information density.

💫 Frictionless Contribution

Rating and adding data happens in-context through hover/swipe interactions.

Core Interactions

  • Global View Toggle: Simple/Detailed switch affects all cards
  • Per-Card Toggle: Click any card to toggle its individual view
  • Hover-to-Rate: Desktop users hover over rating container to reveal stars
  • Swipe-to-Rate: Mobile users swipe left on cards to access rating
  • Distribution Details: Mobile users tap 📊 icon for full breakdowns
  • Inline Contribution: Add side effects and other data without leaving the page

Distribution Display Logic

// Format distribution data for display const formatDistribution = (field, responses) => { // Sort by count descending const sorted = responses.sort((a, b) => b.count - a.count); // Take top 3 + calculate "others" const top3 = sorted.slice(0, 3); const others = sorted.slice(3); const othersPercent = others.reduce((sum, r) => sum + r.percentage, 0); // Format for display return { values: top3.map(r => ({ label: r.value, percentage: r.percentage })), others: othersPercent > 0 ? othersPercent : null, totalReports: responses.reduce((sum, r) => sum + r.count, 0) }; }; // Display component const DistributionField = ({ label, distribution }) => { const isStacked = distribution.values.length > 3 || distribution.values.some(v => v.label.length > 20); return ( <div className="distribution-field"> <div className="distribution-label"> {label} <span className="field-count">({distribution.totalReports})</span> </div> <div className={`distribution-values ${isStacked ? 'stacked' : ''}`}> {distribution.values.map((value, index) => ( <span key={index} className="distribution-value"> {index > 0 && !isStacked && '• '} {value.label} <span className="distribution-percent">({value.percentage}%)</span> </span> ))} {distribution.others && ( <span className="distribution-others"> {!isStacked && '• '}others <span className="distribution-percent">({distribution.others}%)</span> </span> )} </div> </div> ); };

6. Edge Cases & Special Handling

Low Sample Size (n ≤ 3)

Display Pattern
FREQUENCY (3)
Once daily (67%) • Twice daily (33%)

Show all values when n ≤ 3, don't use "others" grouping

AI-Seeded Data (n = 1)

Display Pattern
TIME TO RESULTS (1)
3-4 weeks

No percentages shown, just the single value

Multi-Select Fields (Side Effects)

Special Handling

Each option shows percentage of people who experienced it:

Nausea (61%) Insomnia (48%) None reported (22%)

Percentages don't sum to 100% for multi-select fields
